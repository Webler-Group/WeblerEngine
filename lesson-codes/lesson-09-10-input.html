Lesson 8 — Input

Three demos: Action Bindings, Canvas Controls (on-screen joystick & button), and Pointer Position (PointerPosition maps screen coords to world space via screenToWorld — works correctly with a moving camera).

WASD / arrows — move
Space / FIRE — shoot
Touch joystick or FIRE button on canvas  ·  Desktop: WASD + Space
Click or tap anywhere on the canvas — a marker appears at the world position. Watch the markers stay fixed as the camera drifts.

Action Bindings

InputManager translates raw device events into named actions. Multiple devices can bind to the same action.

move.x0.00
move.y0.00
fire—

Action event log:

input.addBinding('move', { device: 'keyboard', control: 'dpad', }); input.onAction('move', 'start', v => log(v)); input.onAction('move', 'change', v => log(v)); input.onAction('move', 'end', () => log('—'));

Canvas Controls

PointerJoystick and Button are InputControls drawn on the canvas. They claim pointer events by hit-testing in screen-pixel space.

joystick.y0.00
joystick activefalse
button pressedfalse
// Screen-space position (px) const js = new PointerJoystick( ptrDev, { radius: 55 }); js.position.set(85, 335); const btn = new Button(ptrDev, { shape: 'circle', radius: 35 }); btn.position.set(475, 335); // Visualise with custom drawables scene.addNode(new JoystickDraw(js)); scene.addNode(new ButtonDraw(btn));

Pointer Position
